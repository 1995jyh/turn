프로젝트를 진행하는 동안 항상 아래의 원칙을 지켜서 작업해야 해

1. 스킬 장착 (Context Loading): > 개발 하기 전 /skills 폴더 하위에 있는 4개의 폴더를 순회하면서, 각각의 SKILL.md 파일을 읽고 너의 시스템 메모리에 규칙을 숙지하는 것을 까먹지 말 것

2. 개발 규칙 (Development): > 앞으로 코드를 짤 때는 반드시 nextjs-app-router-patterns와 typescript-expert의 지침을 최우선으로 적용해서, 중복 없고 타입이 안전한 코드를 작성해.

3. 품질 검증 (QA): > 코드를 작성하거나 수정한 직후에는 항상 code-reviewer 모드로 전환해서 스스로 버그나 접근성, 성능 문제가 없는지 자체 검수(Self-Review)를 진행해.

4. 필수 강제 사항: > > 너는 코드를 작성하거나 수정한 후, 답변을 종료하기 전에 무조건 마지막에 [히스토리 업데이트 요약] 이라는 항목을 출력해야 해. > > 이 요약을 출력함과 동시에 doc-coauthoring 스킬을 발동하여 /history/history.md 파일에 오늘 개발한 내용과 결정 사항을 실제로 기록하는 파일 수정 작업을 진행해. 히스토리 파일 수정이 완료되지 않으면 이 작업은 끝난 것이 아니야.

5. 모든 답변은 한국어로 진행, 코드 작성 시에는 주석을 한글로 필수로 달 것

6. 중복 코딩, 중복 생성을 방지하며 항상 최적화를 생각할 것
