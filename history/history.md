## 2026-03-08

### Phase 1-2: 기본 UI 및 로비 뼈대 구축 완료
- **초기 랜딩 페이지(`app/page.tsx`) 구축:**
  - 끄투(kkutu.kr) 스타일의 테마(산뜻한 배경, 도트 패턴) 적용
  - 비스듬하게 배치된 타이틀 로고 영역과 4종 소셜 로그인(임시 버튼) 배치
  - '나무 표지판' CSS 컴포넌트(`WoodenButton`)를 제작하여 직관적이고 게임스러운 입장 버튼 구현
- **로비 레이아웃 및 컴포넌트 통합 (`app/lobby/layout.tsx`):**
  - **헤더(`LobbyHeader`)**: 좌우 분리형 네비게이션 적용 (좌측엔 메인 로고, 우측엔 알림/상점 설정 아이콘)
  - **사이드바(`LobbySidebar`)**: 향후 기능 확장성(순위전, 빙고, 사전 등)을 고려한 좌측 메뉴 구축
  - **임시 패널(`LobbyRightPanel`)**: 우측 300x250 광고 배너 영역 및 비로그인 유저 리디렉션용 안내 공간 마련
- **로비 메인 화면 (`app/lobby/page.tsx`) 개편:**
  - 화면 중앙 집중도를 높이기 위해, CSS 기반 임시 젤리 캐릭터 렌더링
  - 메인 캐릭터 하단 닉네임, 레벨 바, 계정 재화(보석/코인) 영역을 이관하여 유저 스탯 확인 용이성 제고
  - 대형 메인 '빠른 시작' 액션 버튼 설계 및 위치 최적화
  - **[UI 고도화]** 상단 헤더의 텍스트 메뉴를 제거하고 화면 중앙의 메인 캐릭터 위에 4종 메인 게임(끝말잇기, 초성퀴즈, 넌센스, 그림맞추기) 진입 버튼을 카드 형태로 큼직하게 배치
  - **[라우팅 및 동적 렌더링]** 메인 로비 컴포넌트를 `BaseLobbyContent`로 분리하여 재사용성 확보. 각 게임별 라우팅(`/lobby/[game]`) 시 배경색과 게임 진입 버튼의 활성화 상태(크기 및 Glow 이펙트)가 동적으로 연동되도록 구현
  - **[하단 액션 메뉴 추가]** 기존 '빠른 시작' 버튼의 크기를 줄이고, 하단에 4종 기능 카드("랜덤 매칭", "방 생성", "커스텀 입장", "방 목록")를 Grid 구조로 한 눈에 보이게 추가 배치

### Phase 1-4: 로비 방 목록(Room List) UI 임시 구현 및 레이아웃 조정
- **방 목록 패널 컴포넌트(`RoomListPanel.tsx`) 추가:**
  - 사용자가 제시한 예시 이미지 기반으로 임시 레이아웃 디자인 (방 번호, 방 제목, 옵션, 라운드, 참가자 수 등)
  - 더미 데이터를 활용해 현재 접속한 게임 모드(URL)에 따라 방 목록 필터링 기능 구현
  - 방 목록을 2줄(Grid, 2-column)로 배치하고, 패널의 세로 길이를 늘려 더 많은 해상도를 활용하도록 UI 튜닝 (사용자 요청 반영)
- **로비 메인 레이아웃 통합 (`BaseLobbyContent.tsx`):**
  - 상단의 게임 선택 4개 메뉴의 위치를 더 위쪽으로 상향 조정하여 하단 방 목록 패널이 차지할 수 있는 시각적 공간 확보
  - 하단의 '방 목록' 버튼 클릭 시, 중앙의 캐릭터/유저 프로필 영역이 방 목록 패널로 전환(토글)되도록 상태(State) 로직 추가
  - 부드러운 전환을 위한 애니메이션 효과 적용

### Phase 1-5: 개별 게임 대기방(Room) UI 임시 구현 및 라우팅
- **게임방 공통 레이아웃 컴포넌트(`GameRoomLayout.tsx`) 제작:**
  - 하위 메뉴들(끝말잇기, 넌센스 등)에 공통으로 적용될 `GameRoomLayout` 제작, 각 게임 특색에 맞는 배경색 동적 반영
  - 상단 방 정보 표시(방 번호, 게임명, 제목) 및 우측 상단 '나가기' 버튼 구현
- **4종 게임의 임시 대기방 페이지 생성:**
  - `app/room/kkutu/[id]/page.tsx`: 끝말잇기용 대기방
  - `app/room/initial/[id]/page.tsx`: 초성 퀴즈용 대기방
  - `app/room/nonsense/[id]/page.tsx`: 넌센스용 대기방
  - `app/room/catchmind/[id]/page.tsx`: 그림맞추기용 대기방 (캔버스 영역 표시)
  - 각 스크린엔 8인의 더미 유저 슬롯과 채팅/액션 영역이 포함되도록 뼈대 구조를 잡음
- **방 목록 컴포넌트(`RoomListPanel.tsx`) 연결:**
  - 표시된 방 카드를 클릭 시 `Link` 컴포넌트를 이용해 해당 게임 타입과 방 번호(id)에 맞는 동적 라우팅 주소로 이동하도록 로직 구현

### Phase 1-6: 로비 및 방 통합 UI 프로토타입 구현 (넌센스 방 기준)
- **대기/게임 통합 UI 디자인 시안 임시 적용:**
  - `app/room/nonsense/[id]/page.tsx`에 신규 레이아웃 반영
  - **상단/중단 영역**: 게임 진행 영역(문제 표시란, 게임 텍스트 등)
